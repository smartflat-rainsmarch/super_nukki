# UI 이미지 → PSD 변환 SaaS 기획서

## 1. 프로젝트 개요

- **가칭 서비스명**: UI2PSD Studio / Screen2PSD / PixelSplit AI
- **목표**: 사용자가 웹에서 단일 UI 스크린샷(앱 화면, 웹 페이지 캡처, AI 생성 화면 등)을 업로드하면,
  1. 화면을 분석하고
  2. 버튼 / 텍스트 / 아이콘 / 배경 / 카드 / 입력창 / 이미지 / 섹션 등을 자동 분리하고
  3. 필요한 경우 가려진 배경을 자연스럽게 복원하고
  4. 각 요소를 레이어화하여
  5. 최종적으로 **PSD 파일로 다운로드**할 수 있게 하는 서비스
- **핵심 가치**:
  - AI로 생성된 “통이미지”를 실제 개발/디자인에 재사용 가능하게 변환
  - 개발자/디자이너의 수작업 복원 시간을 대폭 절감
  - 프로토타입 → 실제 제작 전환 시간을 줄임

---

## 2. 이 서비스가 돈이 되는 이유

### 2.1 실제 수요가 큰 사용자군
1. **프론트엔드 개발자**
   - AI가 뽑아준 UI 시안을 실제 컴포넌트 단위로 해석하고 싶음
2. **디자이너**
   - PNG/JPG 시안을 PSD/Figma 레이어 구조로 복원하고 싶음
3. **에이전시/외주업체**
   - 고객이 보낸 캡처본을 빠르게 편집 가능한 형태로 바꾸고 싶음
4. **스타트업 PM/기획자**
   - 생성형 AI 결과물을 바로 수정/개발 가능한 구조로 넘기고 싶음
5. **마케터/운영자**
   - 홍보 배너/앱 상세페이지용 이미지에서 구성요소를 재활용하고 싶음

### 2.2 고객이 실제로 사는 가치
- 포토샵에서 레이어를 다시 따는 시간 절약
- 배경 제거/텍스트 추출/아이콘 분리/버튼 재구성 자동화
- 개발 가능한 리소스화
- “AI로 만든 예쁜 이미지”를 “실무용 산출물”로 전환

### 2.3 차별점
기존 배경제거 서비스와 다르게, 이 서비스는 단순 컷아웃이 아니라:
- **UI 요소 구조 이해**
- **텍스트 OCR + 스타일 추정**
- **레이어 재구성**
- **빈 공간 인페인팅**
- **PSD 레이어 패키징**
을 하나의 파이프라인으로 제공해야 함

## 3. 제품 비전

### 3.1 한 줄 정의
> “AI가 만든 한 장의 UI 이미지를, 실무에서 편집 가능한 PSD 레이어 파일로 되돌리는 서비스”

### 3.2 장기 비전
단순 PSD 복원기를 넘어서 다음까지 확장 가능:
- PSD 내보내기
- Figma 내보내기
- HTML/CSS 초안 생성
- React/Vue 컴포넌트 스켈레톤 생성
- 디자인 토큰(color, radius, spacing, typography) 추출
- UI 컴포넌트 라이브러리 자동 생성

## 4. 핵심 문제 정의

사용자가 업로드한 UI 스크린샷에는 아래 문제가 있음.

1. **텍스트가 픽셀로 합쳐져 있음**
2. **버튼과 배경이 붙어 있음**
3. **카드/섹션/이미지/아이콘이 한 장으로 병합되어 있음**
4. **텍스트를 제거하면 배경이 비어버림**
5. **원본 폰트/스타일/레이어 순서를 알 수 없음**
6. **그림자/블러/그라디언트/투명도 등 복합 효과가 포함될 수 있음**
7. **AI 생성 이미지 특성상 실제 UI 규칙과 다를 수 있음**

따라서 이 서비스는 단순 CV가 아니라,
**비전 + OCR + 세그멘테이션 + 인페인팅 + 레이어 엔지니어링 + PSD 생성기**가 결합된 시스템이어야 함.

## 7. 가장 중요한 기술 난제

### 7.1 “화면 이해” 문제
그냥 객체 탐지로는 부족함. UI는 일반 사진과 달리:
- 직사각형 레이아웃이 많고
- 텍스트 비중이 크며
- 정렬 규칙이 있으며
- 반복 컴포넌트가 많음

즉 일반 객체탐지 모델 + UI 특화 후처리가 필요.

### 7.2 “텍스트 제거 후 배경 복원” 문제
텍스트를 제거하면 뒤 배경이 비어버리므로,
- 단색 배경이면 쉬움
- 그라디언트면 중간 난이도
- 이미지 배경이면 어려움
- 카드 그림자/블러/패턴이 있으면 매우 어려움

### 7.3 “PSD 레이어 품질” 문제
진짜 돈을 받으려면 결과물이 “그럴듯한 PSD”가 아니라:
- 정리된 레이어명
- 그룹 구조
- 편집 가능한 텍스트
- 누락 적은 배경
- 에셋 재사용 가능성
이 있어야 함.

### 7.4 “AI 생성 UI” 특유의 비정형성
생성형 AI UI는 종종:
- 버튼 모서리 일관성이 없음
- 폰트가 비현실적임
- 아이콘이 미묘하게 찌그러짐
- 그림자 규칙이 비정상적임

그래서 “완전 자동”보다 **자동 + 보정 UX**가 핵심 경쟁력.

## 9. 추천 엔진 아키텍처

### 9.1 파이프라인 구조

```text
[업로드]
   ↓
[전처리]
   ↓
[UI 레이아웃 감지]
   ↓
[텍스트/아이콘/버튼/카드 분리]
   ↓
[OCR + 스타일 추정]
   ↓
[제거 후 배경 인페인팅]
   ↓
[레이어 후처리 / 그룹핑]
   ↓
[PSD 빌더]
   ↓
[미리보기 + 다운로드]
```

### 9.2 모듈별 구성

#### A. Preprocess Engine
- 해상도 정규화
- 노이즈 제거
- 샤프닝/대비 보정
- 스케일 분석
- 디바이스 프레임/여백 제거

#### B. Layout Engine
- 큰 영역 분리
- 컴포넌트 블록 경계 탐지
- Z-order 추정
- 반복 패턴 감지

#### C. OCR Engine
- 텍스트 박스 감지
- 텍스트 추출
- 줄바꿈 구조 추정
- 폰트 속성 근사

#### D. Segmentation Engine
- 버튼/카드/이미지/아이콘/배경 분리
- 마스크 생성
- 경계 보정
- 알파 추출

#### E. Inpainting Engine
- 텍스트 제거용 마스크 생성
- 배경 보정
- 복원 품질 점수 계산

#### F. Layer Composer
- 요소별 PNG 생성
- 레이어 좌표 기록
- 그룹 구조 생성
- export manifest 생성

#### G. PSD Builder
- 캔버스 생성
- 레이어 삽입
- 그룹 구조 삽입
- 텍스트 가능 시 텍스트화
- 불가능 시 raster fallback

**“모바일 UI 스크린샷 1장 → 실용적인 PSD 초안 생성”**

- PSD export
- 에셋 PNG export
- 작업 결과 미리보기

### 10.3 MVP에서 빼도 되는 것
- 웹 페이지 전체 스크롤 캡처 완벽 지원
- 복잡한 일러스트 분리
- 고급 그림자/블렌딩 완벽 복원

## 14. 모델/엔진 상세 추천

### 14.1 OCR
#### 우선순위
1. PaddleOCR
2. EasyOCR
3. 상용 OCR 옵션(프로 플랜)

#### 이유
- UI 텍스트는 일반 문서 OCR과 달리 작은 글씨, 버튼 라벨, 흐린 회색 텍스트가 많음
- 복수 엔진 앙상블이 유리함

### 14.2 세그멘테이션
#### 우선순위
1. SAM 2 기반 범용 분리
2. UI 전용 후처리 rule engine
3. Detectron2 기반 커스텀 컴포넌트 검출

#### 이유
- 범용 분리는 SAM 계열이 강력하지만 UI의 의미 구조까지는 부족할 수 있음
- 따라서 “범용 분리 + UI 규칙엔진” 조합이 현실적

### 14.3 인페인팅
#### 우선순위
1. OpenCV inpainting (빠름, MVP용)
2. LaMa류 (품질 향상)
3. Diffusion inpainting (고급 옵션)

#### 이유
- 초기에 모든 요청을 diffusion으로 처리하면 비용/속도가 불리함
- 무료/일반은 빠른 엔진, 프로는 고품질 엔진 이원화가 좋음

### 14.4 PSD 빌더
- 1차: PNG 레이어 묶음 중심 PSD 생성
- 2차: 텍스트 레이어/그룹 구조 강화
- 3차: 블렌드/마스크/효과 보존 강화

## 28. 최종 결론

이 아이템은 단순 이미지 편집 툴이 아니라,
**“AI 생성 결과물을 실무 산출물로 변환하는 브릿지 제품”** 이라는 점에서 충분히 사업성이 있음.

다만 성공하려면 아래 원칙이 중요함:

1. 처음부터 완벽 복원을 약속하지 말 것
2. 모바일 UI 한정 MVP로 시작할 것
3. 자동화 + 수동 보정 UX를 함께 제공할 것
4. PSD 품질을 가장 중요한 가치로 둘 것
5. 무료 체험은 주되, 가격은 너무 낮게 잡지 말 것

가장 추천하는 시작 전략은 다음과 같음:

- **MVP 타깃**: 모바일 앱 UI 스크린샷
- **핵심 기능**: OCR + 버튼/카드/배경 분리 + 텍스트 제거 후 인페인팅 + PSD export
- **기술 스택**: Next.js + FastAPI + PyTorch + OpenCV + PaddleOCR + SAM 2
- **가격 정책**:
  - 무료: 월 3회
  - 일반: 월 $39
  - 프로: 월 $129

이 구조라면 개발자 대상 B2C/B2B 사이의 좋은 SaaS 포지션을 만들 수 있음.
