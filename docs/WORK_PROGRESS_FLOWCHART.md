# UI2PSD Studio - 작업 진행순서도

> 최종 수정일: 2026-03-31
> 총 개발 기간: 12주 (Phase 1~6)
> 원칙: MVP 우선, 한 번에 하나의 기능만

---

## 전체 흐름 개요

```
Phase 1: 프로젝트 초기 세팅 (1~2주차)
    ↓
Phase 2: 핵심 엔진 프로토타입 (3~4주차)
    ↓
Phase 3: 웹 MVP 구현 (5~6주차)
    ↓
Phase 4: 사용자 시스템 + 결제 (7~8주차)
    ↓
Phase 5: 품질 향상 + 관리 (9~10주차)
    ↓
Phase 6: 베타 출시 준비 (11~12주차)
```

## Phase 1: 프로젝트 초기 세팅 (1~2주차)

### 목표
레포 구조 생성, 개발 환경 구축, 업로드/저장 파이프라인

### 작업 순서

```
1.1 레포 구조 생성
 ├─ apps/web/       (Next.js)
 ├─ apps/api/       (FastAPI)
 ├─ apps/worker/    (Celery)
 ├─ storage/        (uploads, outputs, temp, debug)
 └─ docs/
     ↓
1.2 Docker Compose 작성
 ├─ web (Next.js)
 ├─ api (FastAPI)
 ├─ worker (Celery)
 ├─ redis
 └─ postgres
     ↓
1.3 FastAPI 프로젝트 뼈대
 ├─ 라우터 구조 (upload, project, download, usage)
 ├─ DB 연결 (SQLAlchemy + PostgreSQL)
 └─ Redis 연결
     ↓
1.4 Next.js 프로젝트 뼈대
 ├─ 페이지 라우팅 (/, /upload, /project/[id], /pricing)
 ├─ Tailwind + shadcn/ui 설정
 └─ API 클라이언트 설정
     ↓
1.5 DB 스키마 생성
 ├─ users 테이블
 ├─ projects 테이블
 ├─ layers 테이블
 ├─ jobs 테이블
 └─ billing 테이블
     ↓
1.6 파일 업로드 파이프라인
 ├─ POST /api/upload 구현
 ├─ S3/로컬 스토리지 저장
 ├─ project 레코드 생성
 └─ 프론트 업로드 UI (Drag & Drop)
```

### 산출물
- [x] 실행 가능한 Docker 환경
- [x] 이미지 업로드 → 저장 → project_id 반환 동작

### 검증 기준
- `docker compose up` 성공
- 이미지 업로드 후 storage에 파일 존재 확인
- DB에 project 레코드 생성 확인

## Phase 2: 핵심 엔진 프로토타입 (3~4주차)

### 목표
OCR, 세그멘테이션, 인페인팅, PSD 생성 기술 검증

### 작업 순서

```
2.1 전처리 엔진 (Preprocess)
 ├─ 해상도 정규화
 ├─ 노이즈 제거 / 샤프닝
 └─ 디바이스 프레임 제거
     ↓
2.2 OCR 엔진 연결
 ├─ PaddleOCR 설치 + 연동
 ├─ 텍스트 박스 감지
 ├─ 텍스트 추출
 └─ 폰트 크기/색상/정렬 추정
     ↓
2.3 세그멘테이션 엔진
 ├─ SAM 2 모델 로드
 ├─ UI 요소 분리 (버튼/카드/아이콘/배경)
 ├─ 마스크 생성
 └─ UI 전용 후처리 규칙엔진
     ↓
2.4 인페인팅 엔진 (MVP)
 ├─ 텍스트 영역 마스크 생성
 ├─ OpenCV inpainting 적용
 └─ 복원 결과 품질 점수 산출
     ↓
2.5 레이어 Composer
 ├─ 요소별 PNG 생성
 ├─ 레이어 좌표 기록
 └─ 그룹 구조 생성 (Header/Card/CTA 등)
     ↓
2.6 PSD Builder 프로토타입
 ├─ psd-tools 기반 캔버스 생성
 ├─ PNG 레이어 삽입
 ├─ 그룹 구조 삽입
 └─ 텍스트 레이어 or raster fallback
```

### 산출물
- [x] 모바일 UI 이미지 1장 → PSD 변환 성공

### 검증 기준
- OCR: 텍스트 추출 정확도 70%+ (모바일 UI 기준)
- 세그멘테이션: 주요 요소 5개 이상 분리
- 인페인팅: 텍스트 제거 후 배경 자연스러움 확인
- PSD: Photoshop에서 열림 + 레이어 확인

## Phase 3: 웹 MVP 구현 (5~6주차)

### 목표
업로드 → 처리 → 미리보기 → 다운로드 전체 플로우

### 작업 순서

```
3.1 Celery Worker 연동
 ├─ Job 큐 등록 (Redis)
 ├─ Worker가 파이프라인 실행
 └─ Job 상태 업데이트 (queued→processing→completed/failed)
     ↓
3.2 처리 상태 API
 ├─ GET /api/project/{id} (status, progress %)
 ├─ WebSocket 또는 Polling 기반 실시간 상태
 └─ 단계별 진행률 표시
     ↓
3.3 결과 API
 ├─ GET /api/project/{id}/result (psd_url, layers[])
 ├─ GET /api/download/{id} (PSD 파일)
 └─ 에셋 ZIP 다운로드
     ↓
3.4 업로드 페이지 UI
 ├─ Drag & Drop 영역
 ├─ 클립보드 붙여넣기
 ├─ 파일 선택기
 └─ 업로드 진행률
     ↓
3.5 처리 대기 페이지 UI
 ├─ 단계별 진행 표시 (전처리→분석→분리→인페인팅→PSD)
 ├─ 예상 처리 시간
 └─ 처리 중 썸네일 미리보기
     ↓
3.6 결과 미리보기 페이지 UI
 ├─ 원본 vs 분리 결과 Before/After 비교
 ├─ 레이어 목록 표시
 ├─ 텍스트 추출 목록
 ├─ PSD 다운로드 버튼
 └─ "다시 처리" 버튼
```

### 산출물
- [x] 웹에서 이미지 업로드 → PSD 다운로드 전체 플로우 동작

### 검증 기준
- 업로드 → 다운로드까지 E2E 테스트 통과
- 처리 시간 30초 이내 (모바일 UI 기준)
- PSD 파일 Photoshop에서 정상 열림

## Phase 4: 사용자 시스템 + 결제 (7~8주차)

### 목표
회원가입/로그인, 요금제, 결제, 사용량 제한

### 작업 순서

```
4.1 인증 시스템
 ├─ 회원가입 (이메일/비밀번호)
 ├─ 로그인 / 로그아웃
 ├─ JWT 토큰 관리
 └─ 프론트 인증 상태 관리
     ↓
4.2 사용량 관리
 ├─ GET /api/usage (remaining 횟수)
 ├─ 플랜별 월간 한도 체크
 ├─ 사용 시 usage_count 차감
 └─ 월 초 reset_date 기준 리셋
     ↓
4.3 요금제 적용
 ├─ 무료: 월 3회, 1024px 제한
 ├─ 일반 ($39): 월 100회, HD
 ├─ 프로 ($129): 월 500회, 4K, API
 └─ 한도 초과 시 업그레이드 유도
     ↓
4.4 Stripe 결제 연동
 ├─ 구독 생성 (월간/연간)
 ├─ 결제 Webhook 처리
 ├─ 플랜 변경/취소
 └─ 추가 크레딧 구매
     ↓
4.5 가격 페이지 UI
 ├─ 플랜 비교표
 ├─ 결제 버튼 → Stripe Checkout
 └─ 현재 플랜 표시
     ↓
4.6 마이페이지 UI
 ├─ 작업 이력 목록
 ├─ 재다운로드
 ├─ 구독 상태 / 잔여 횟수
 └─ 플랜 변경
```

### 산출물
- [x] 무료/유료 사용자 분리 동작
- [x] 결제 → 플랜 업그레이드 동작

### 검증 기준
- 무료 사용자 3회 초과 시 업그레이드 안내
- Stripe 테스트 결제 성공
- 플랜 변경 후 usage 한도 변경 확인

## Phase 5: 품질 향상 + 관리 (9~10주차)

### 목표
엔진 품질 개선, 수동 보정 UI, 관리자 모니터링

### 작업 순서

```
5.1 OCR 품질 향상
 ├─ PaddleOCR + EasyOCR 앙상블
 ├─ 텍스트 confidence 표시
 └─ 작은 글씨 / 흐린 텍스트 보정
     ↓
5.2 세그멘테이션 품질 향상
 ├─ UI 컴포넌트 규칙엔진 강화
 ├─ 반복 컴포넌트 자동 감지
 └─ 마스크 경계 보정
     ↓
5.3 인페인팅 고도화
 ├─ LaMa 엔진 도입 (프로 플랜)
 ├─ 그라디언트/패턴 배경 대응
 └─ 복원 품질 낮은 영역 경고 표시
     ↓
5.4 수동 보정 UI (편집 보정)
 ├─ 자동 탐지 영역 수동 수정 (Konva.js)
 ├─ 잘못 분리된 박스 조절
 ├─ 레이어 병합/분리 옵션
 └─ PSD 생성 전 검토 화면
     ↓
5.5 관리자 페이지
 ├─ 사용자 목록 / 통계
 ├─ 작업 모니터링 (성공/실패율)
 ├─ 시스템 리소스 현황
 └─ 에러 로그 조회
     ↓
5.6 오류 처리 강화
 ├─ 작업 실패 시 재시도 로직
 ├─ 타임아웃 처리
 ├─ 사용자 친화적 에러 메시지
 └─ Sentry 연동
```

### 산출물
- [x] 수동 보정으로 결과물 품질 개선 가능
- [x] 관리자 대시보드에서 서비스 상태 모니터링

### 검증 기준
- 앙상블 OCR 정확도 85%+
- LaMa 인페인팅 적용 시 품질 점수 향상 확인
- 관리자 페이지에서 실시간 작업 현황 확인

 ├─ 서비스 소개 (한 줄 정의)
 ├─ Before/After 데모 이미지

 ├─ 가격 플랜 요약
 └─ CTA 버튼 (무료 체험)
     ↓
6.2 온보딩 플로우
 ├─ 가입 직후 가이드
 ├─ 첫 업로드 유도
 └─ 결과 확인 → 플랜 안내
     ↓
6.3 UX 신뢰 요소 추가
 ├─ "편집 가능한 초안 생성" 안내 문구
 ├─ 텍스트 confidence 표시
 ├─ 인페인팅 품질 낮은 영역 경고
 └─ raster vs editable 레이어 구분 표시
     ↓
6.4 샘플 데이터 검수
 ├─ 모바일 UI 50장 테스트
 ├─ 변환 성공률 확인 (목표: 90%)
 ├─ 결과물 품질 평가
 └─ 엣지 케이스 수집 및 대응
     ↓
6.5 성능 최적화
 ├─ 평균 처리시간 20초 이하 확인
 ├─ Worker 수평 확장 테스트

6.6 베타 출시
 ├─ 도메인 + SSL 설정
