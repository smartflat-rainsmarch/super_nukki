# 이메일 인증번호 회원가입 기획서

> 작성일: 2026-04-01

## 1. 목적
이메일 회원가입 시 입력한 이메일로 6자리 인증번호를 발송하여 이메일 소유권을 검증한 후에만 가입을 허용한다.

## 2. 회원가입 플로우

```
[1단계] 이메일 입력 → "인증번호 받기" 클릭
    → 6자리 인증번호가 입력한 이메일로 발송 (5분 유효)

[2단계] 인증번호 6자리 입력 → "확인" 클릭
    → 인증번호 일치 시 다음 단계로
    → 불일치 시 에러 표시 + 재입력
    → 만료 시 "인증번호 재발송" 버튼

[3단계] 비밀번호 + 비밀번호 확인 입력 → "회원가입" 클릭
    → 가입 완료 → 자동 로그인 → /upload 이동
```

## 3. API 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| POST | `/api/auth/send-code` | 인증번호 생성 + 이메일 발송 |
| POST | `/api/auth/verify-code` | 인증번호 검증 → verified_token 반환 |
| POST | `/api/auth/register` | 회원가입 (verified_token 필수) |

## 4. 이메일 템플릿

```
제목: [UI2PSD] 인증번호: 123456

본문:
UI2PSD Studio 회원가입 인증번호

123456

이 인증번호는 5분간 유효합니다.
본인이 요청하지 않았다면 이 이메일을 무시하세요.
```

## 5. 보안 규칙

| 규칙 | 값 |
|------|-----|
| 인증번호 길이 | 6자리 숫자 |
| 유효 시간 | 5분 |
| 동일 이메일 재발송 간격 | 60초 |
| 최대 시도 횟수 | 5회 (초과 시 잠금) |
| verified_token 유효 시간 | 10분 |

## 6. 에지 케이스

| 상황 | 동작 |
|------|------|
| 이미 가입된 이메일 | "이미 가입된 이메일입니다" 에러 |
| 소셜 가입된 이메일 | "Google/카카오로 가입된 계정입니다" 에러 |
| 인증번호 만료 | "인증번호가 만료되었습니다. 다시 받기" |
| 5회 틀림 | "너무 많은 시도입니다. 잠시 후 다시 시도하세요" |
| SMTP 미설정 (개발) | API 응답에 인증번호 포함 (dev_code 필드) |
